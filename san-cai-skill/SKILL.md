# 三才处境分析 Skill

## 概述

这是一套基于《易经》八卦和三才框架的**伦理处境分析工具**。核心逻辑：

```
三才位置（地/人/天） × 阴阳状态（阳爻/阴爻） × 正负发挥方向（正面/负面）
= 3 × 2 × 2 = 12 个基础概念单元
= 每卦 8 种解读 → 八卦合计 64 种处境
```

**核心前提**：卦象是地形图，不是判决书。吉凶不取决于阴阳本身，而取决于你在每个爻位上的**选择**。

## 用户流程（7 步）

整体节奏：**一次只问一件事，信息充分了再推进**。每轮对话控制在一个卡片组 + 一个引导问题的范围内，几分钟能讲完。轮次是自然的对话节律，不需严格计数——信息不充分时就追问，充分了就往前走。

```
用户描述处境
  ↓
Step 1: 选定并缩窄场景          → steps/01-选定场景.md
    第一轮：确定大领域
    第二轮：缩窄到具体场景
  ↓
Step 2: 三才隐喻与场景结合       → steps/02-三才映射.md
    第一轮：地位映射 → 用户确认 → 口头小结
    第二轮：人位映射 → 用户确认 → 口头小结
    第三轮：天位映射 → 用户确认 → 口头小结
    三轮完成后输出三才定位结果
  ↓
Step 3: 阴阳质性映射（中性）     → steps/03-阴阳定性.md
    第一轮：地位阴阳 → 用户辨认 → 口头小结
    第二轮：人位阴阳 → 用户辨认 → 口头小结
    第三轮：天位阴阳 → 用户辨认 → 口头小结
```

## 核心原则

1. **阴阳不是好/坏**，是两种质性（向外/向内、刚健/柔顺）
2. **吉凶在时位**，不在阴阳本身
3. **概念永远宽于例子**——先感受隐喻场，再看具体例子
4. **信息充分性原则**：在每一步推进之前，AI 必须检查当前拥有的信息是否足以支撑下一步。关键信息模糊、矛盾或缺失时，优先追问而非猜测推进。追问最多两轮，仍不清晰则标记为"模糊"带入下一步，并在输出中注明。
5. **引导式追问原则**：追问时优先采用选择题/问题卡片方式，降低用户的认知负担。提供 3-5 个选项让用户辨认，而不是只抛开放式问题。
6. **逐位置、小批量原则**：在同一步骤内，不一次性抛出多个位置的问题。每个位置独立对话，一次只给一个卡片组。信息充分了即确认推进，信息不充分则追问。确认后的口头小结要轻——一句话确认即可，不展开长篇。回退局部化（"都不準"只影响当前对话，不阻塞其他位置）。

## 追问机制与信息充分性检查

### 信息充分性检查标准

在每个步骤推进到下一步之前，AI 需检查以下通用维度：

| 维度 | 含义 | 判断标准 |
|:--|:--|:--|
| **明确** | 用户对该信息给出了具体的、非模糊的回答 | 不是"我也不知道""可能吧"这类模糊表达 |
| **一致** | 用户在不同问题上的回答没有明显矛盾 | 如果有矛盾，优先澄清而非强行解读 |
| **完整** | 该步骤所需的关键信息维度都已覆盖 | 具体标准见各步骤文件的"信息充分性检查"段 |

三个维度都满足 → 推进。有任一不满足 → 启动追问。

### 追问规则

1. **优先用问题卡片**：每次追问提供 3-5 个选项，让用户从中辨认，而不是让用户从零描述
2. **最多两轮**：同一问题追问不超过两轮。两轮后仍不清晰 → 标记为"模糊"，带标记推进
3. **标记传递**：模糊标记随输出带入下一步，后续步骤在解读时注明"用户在此位置上的信息尚不充分"
4. **自然隐喻兜底**：用户面对抽象概念（阴阳、正负）不知如何回答时，退回到自然隐喻或身体感受层面引导

### 问题卡片标准格式

当需要追问用户时，使用以下格式：

```
**卡片 A：[一句话标题]**
[2-3 句描述，让用户感受到"这就是我的情况"]
> 👉 [直接问用户的问题]

**卡片 B：[一句话标题]**
...

**卡片 Z：都不在以上**
> 👉 或者你的理解不在上面——告诉我你的感觉。
```
